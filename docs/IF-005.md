### Archivo 1: `src/analyxa/config.py`

Configuración centralizada. Single source of truth para settings.

**Clase: `AnalyxaConfig`**

```
__init__(self, env_file: str | Path | None = None)
  - Si env_file se provee → cargar con dotenv.load_dotenv(env_file)
  - Si env_file es None → intentar cargar .env del cwd, silenciosamente ignorar si no existe
  - Después de cargar .env, todos los valores se leen de os.environ con defaults
```

**Propiedades (todas con @property, read-only):**

| Propiedad | Env var | Default | Tipo |
|-----------|---------|---------|------|
| `anthropic_api_key` | ANTHROPIC_API_KEY | None | str \| None |
| `openai_api_key` | OPENAI_API_KEY | None | str \| None |
| `default_provider` | ANALYXA_PROVIDER | "anthropic" | str |
| `default_model` | ANALYXA_MODEL | None (usa default del provider) | str \| None |
| `default_schema` | ANALYXA_SCHEMA | "universal" | str |
| `embedding_model` | ANALYXA_EMBEDDING_MODEL | "text-embedding-3-small" | str |
| `enable_embeddings` | ANALYXA_EMBEDDINGS | True | bool |
| `schemas_dir` | ANALYXA_SCHEMAS_DIR | None (usa default del paquete) | str \| None |
| `redis_url` | REDIS_URL | "redis://localhost:6379" | str |
| `qdrant_url` | QDRANT_URL | "http://localhost:6333" | str |
| `qdrant_collection` | QDRANT_COLLECTION | "analyxa_analyses" | str |
| `log_level` | ANALYXA_LOG_LEVEL | "INFO" | str |

**Notas de implementación:**
- `enable_embeddings` convierte string a bool: "true", "1", "yes" → True, todo lo demás → False
- Las API keys retornan None si no están configuradas (no error — el componente decide qué hacer)
- `schemas_dir` retorna None si no está configurado, lo cual hace que SchemaManager use su default (relativo al paquete)

**Método: `to_dict`**
```
to_dict(self) -> dict
  - Retorna dict con todos los settings (NUNCA incluir API keys completas)
  - API keys: mostrar solo últimos 4 caracteres, ej: "sk-...a1b2" o None
  - Para debug/logging
```

**Método de clase: `from_dict`**
```
@classmethod
from_dict(cls, overrides: dict) -> "AnalyxaConfig"
  - Crear config con override manual de valores
  - Útil para testing y uso programático
  - Setea los values como env vars temporales y luego crea la instancia
```

**Singleton pattern:**
```python
_default_config: AnalyxaConfig | None = None

def get_config(env_file: str | None = None) -> AnalyxaConfig:
    """Retorna config singleton. Crea si no existe."""
    global _default_config
    if _default_config is None:
        _default_config = AnalyxaConfig(env_file=env_file)
    return _default_config

def reset_config():
    """Reset singleton. Útil para tests."""
    global _default_config
    _default_config = None
```

**Imports:** `os`, `pathlib.Path`, `dotenv`

**Comando: `analyze`**

```python
@main.command()
@click.argument("file", type=click.Path(exists=True))
@click.option("--schema", "-s", default="universal", help="Schema to use for analysis")
@click.option("--output", "-o", default=None, help="Output file path (default: stdout)")
@click.option("--format", "-f", "output_format", type=click.Choice(["json", "compact"]), default="json", help="Output format")
@click.option("--provider", "-p", default=None, help="LLM provider (anthropic/openai)")
@click.option("--model", "-m", default=None, help="LLM model override")
@click.option("--no-embeddings", is_flag=True, help="Disable embedding generation")
@click.option("--context", "-c", multiple=True, help="Context key=value pairs (repeatable)")
def analyze(file, schema, output, output_format, provider, model, no_embeddings, context):
    """Analyze a conversation file and extract structured data."""
```

**Lógica del comando analyze:**

1. **Cargar config:**
   ```python
   from analyxa.config import get_config
   config = get_config()
   ```

2. **Parsear context:**
   ```python
   # --context agent_name=SupportBot --context company=Acme
   ctx = {}
   for item in context:
       if "=" in item:
           key, value = item.split("=", 1)
           ctx[key] = value
   context_dict = ctx if ctx else None
   ```

3. **Leer conversación:**
   ```python
   from analyxa.sources.file_source import FileSource
   source = FileSource(file)
   conversation = source.read()
   ```

4. **Crear analyzer y ejecutar:**
   ```python
   from analyxa.analyzer import Analyzer
   
   analyzer = Analyzer(
       schema_name=schema,
       provider=provider or config.default_provider,
       model=model or config.default_model,
       enable_embeddings=not no_embeddings and config.enable_embeddings
   )
   
   result = analyzer.analyze(conversation, context=context_dict)
   ```

5. **Output:**
   ```python
   if output_format == "compact":
       output_data = result.to_flat_dict()
   else:
       output_data = result.to_dict()
   
   json_str = json.dumps(output_data, indent=2, ensure_ascii=False)
   
   if output:
       from analyxa.sinks.json_sink import JsonSink
       sink = JsonSink(output)
       sink.write(output_data)
       click.echo(f"✅ Analysis saved to {output}")
   else:
       click.echo(json_str)
   ```

6. **Error handling:** Envolver todo en try/except:
   - FileNotFoundError → `click.echo("❌ Schema '{schema}' not found.", err=True)` + `sys.exit(1)`
   - ValueError → `click.echo("❌ {error}", err=True)` + `sys.exit(1)`
   - Exception genérica → `click.echo("❌ Analysis failed: {error}", err=True)` + `sys.exit(1)`

```python
@schemas.command("list")
def schemas_list():
    """List available schemas."""
    from analyxa.schema import SchemaManager
    sm = SchemaManager()
    names = sm.list_schemas()
    
    click.echo(f"Available schemas ({len(names)}):\n")
    for name in names:
        schema = sm.load_schema(name)
        meta = schema["metadata"]
        fields = schema["fields"]
        inherits = meta.get("inherits") or "—"
        click.echo(f"  {name}")
        click.echo(f"    Version: {meta['version']}  |  Fields: {len(fields)}  |  Inherits: {inherits}")
        click.echo(f"    {meta['description']}")
        click.echo()
```

**Subcomando: `schemas show`**

```python
@schemas.command("show")
@click.argument("name")
def schemas_show(name):
    """Show detailed schema information."""
    from analyxa.schema import SchemaManager
    sm = SchemaManager()
    
    try:
        schema = sm.load_schema(name)
    except FileNotFoundError:
        click.echo(f"❌ Schema '{name}' not found.", err=True)
        sys.exit(1)
    
    meta = schema["metadata"]
    fields = schema["fields"]
    
    click.echo(f"Schema: {meta['name']} v{meta['version']}")
    click.echo(f"Description: {meta['description']}")
    if meta.get("inherits"):
        click.echo(f"Inherits: {meta['inherits']}")
    click.echo(f"\nFields ({len(fields)}):\n")
    
    for i, field in enumerate(fields, 1):
        req = "required" if field.get("required") else "optional"
        ftype = field["type"]
        click.echo(f"  {i:2d}. {field['name']} ({ftype}, {req})")
        click.echo(f"      {field['description']}")
        if field.get("allowed_values"):
            click.echo(f"      Values: {', '.join(field['allowed_values'])}")
        click.echo()
```

1. **DISCIPLINA DE FABRICANTE:** Ejecuta esta IF tal cual. No renombres funciones, no agregues comandos extra, no "mejores" la estructura. Si algo parece mal → pregunta antes de cambiar. Los 3 eslabones de la cadena (Arquitecto, Fabricante, Director) dependen de que cada uno cumpla su rol exacto.

2. **El refactor de llm_client.py y embeddings.py es MÍNIMO.** Solo agregar la cadena de fallback config → env var. NO reorganizar imports, NO renombrar variables, NO cambiar la lógica existente. Los 36 tests anteriores deben seguir pasando sin cambios.

3. **pyproject.toml reemplaza setup.py.** No crear setup.py, setup.cfg, ni MANIFEST.in. PEP 621 con setuptools es el estándar moderno.

4. **`pip install -e ".[dev]"` es OBLIGATORIO.** Sin esto, el comando `analyxa` no se registra en el PATH y los tests de CLI fallan. Ejecutar DESPUÉS de crear todos los archivos.

5. **`[tool.setuptools.package-data]` es CRÍTICO.** Sin esto, los schemas YAML no se incluyen en el paquete y SchemaManager falla cuando se instala via pip. Verificar que `analyxa schemas list` muestra los 2 schemas después del install.

6. **`[tool.pytest.ini_options]` simplifica todo.** Con `pythonpath = ["src"]` ya no necesitas `PYTHONPATH=src` antes de pytest. Pero verificar que los tests siguen pasando con ambos métodos.

7. **Click CliRunner es self-contained.** No necesita subprocess ni terminal real. Los tests corren en el mismo proceso. Mockear Analyzer para el test de analyze (no hacer llamadas API reales).

8. **Los imports en cli.py son LAZY.** Importar dentro de las funciones de cada comando, no al top del archivo. Esto hace que `analyxa --help` sea instantáneo sin cargar todo el motor.

9. **config.reset_config() en tests.** Siempre llamar en teardown de tests de config para no contaminar entre tests. Usar fixture con yield o monkeypatch de pytest.
