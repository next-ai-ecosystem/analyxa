  - Leer el archivo completo como texto (UTF-8)

```
read_messages(self) -> list[dict]

      Agent: Hi there

  - Retornar lista de dicts
```

```
metadata(self) -> dict
  - Retornar: {"source": "file", "path": str(self.path), "filename": self.path.name}
```

### Archivo 2: `src/analyxa/embeddings.py`

Generador de vectores semánticos. Usa OpenAI text-embedding-3-small (1536D).

**Clase: `EmbeddingGenerator`**

```
__init__(self, model: str = "text-embedding-3-small", api_key: str | None = None)
  - model: modelo de embeddings (default text-embedding-3-small)
  - api_key: si None, leer de OPENAI_API_KEY
  - Inicializar cliente OpenAI
  - Si no hay API key → self._client = None (degradación silenciosa, NO error)
```

```
generate(self, text: str) -> list[float] | None
  - Si self._client es None → retornar None (no hay API key configurada)
  - Si text está vacío → retornar None
  - Llamar a OpenAI embeddings API:
    response = self._client.embeddings.create(model=self.model, input=text)
  - Retornar response.data[0].embedding (lista de 1536 floats)
  - Si la llamada falla → retornar None (degradación silenciosa, log el error a stderr)
```

```
@property
dimensions(self) -> int:
  - Retornar 1536 (fijo para text-embedding-3-small)
```

**Diseño clave: DEGRADACIÓN SILENCIOSA.** Si no hay API key o si la llamada falla, generate() retorna None. El pipeline continúa sin embeddings. Esto permite que el motor funcione sin OpenAI configurado — los embeddings son opcionales.

Imports: `import os, sys`
Import condicional de openai (mismo patrón que llm_client.py)

### Archivo 5: `src/analyxa/analyzer.py`

**EL ORQUESTADOR PRINCIPAL.** Conecta todo el pipeline.

**Dataclass: `AnalysisResult`**

```python
@dataclass
class AnalysisResult:
    # Campos extraídos por el LLM
    fields: dict                    # JSON con los campos del schema
    
    # Metadata automática
    schema_name: str                # Nombre del schema usado
    schema_version: str             # Versión del schema
    analyzed_at: str                # ISO 8601 timestamp
    analysis_model: str             # Modelo LLM usado
    embedding_model: str | None     # Modelo de embedding (None si no se usó)
    session_length: int             # Número de mensajes en la conversación
    conversation_hash: str          # SHA-256 truncado (16 chars)
    
    # Vector semántico
    embedding: list[float] | None   # Vector 1536D (None si no se generó)
    
    # Metadata de ejecución
    llm_response: LLMResponse       # Respuesta completa del LLM (para debug)
    validation_errors: list[str]    # Errores de validación (vacía si todo OK)
    
    def to_dict(self) -> dict:
        """Retorna dict completo con campos + metadata + embedding info.
        Para almacenamiento en Qdrant o exportación completa."""
        result = {}
        result.update(self.fields)
        result["_meta"] = {
            "schema_name": self.schema_name,
            "schema_version": self.schema_version,
            "analyzed_at": self.analyzed_at,
            "analysis_model": self.analysis_model,
            "embedding_model": self.embedding_model,
            "session_length": self.session_length,
            "conversation_hash": self.conversation_hash,
            "has_embedding": self.embedding is not None,
            "input_tokens": self.llm_response.input_tokens,
            "output_tokens": self.llm_response.output_tokens,
            "latency_ms": self.llm_response.latency_ms,
            "validation_errors": self.validation_errors,
        }
        return result
    
    def to_flat_dict(self) -> dict:
        """Retorna dict plano (campos + meta en mismo nivel, sin embedding).
        Para exportación a JSON file, CSV, o display."""
        result = {}
        result.update(self.fields)
        result["schema_name"] = self.schema_name
        result["schema_version"] = self.schema_version
        result["analyzed_at"] = self.analyzed_at
        result["analysis_model"] = self.analysis_model
        result["session_length"] = self.session_length
        result["conversation_hash"] = self.conversation_hash
        return result
```

**Clase: `Analyzer`**

```
__init__(self,
         schema_name: str = "universal",
         provider: str = "anthropic",
         model: str | None = None,
         api_key: str | None = None,
         enable_embeddings: bool = True,
         embedding_api_key: str | None = None)
  
  - Inicializar SchemaManager
  - Cargar el schema (con herencia resuelta)
  - Inicializar LLMClient con provider, model, api_key
  - Si enable_embeddings: inicializar EmbeddingGenerator (con degradación silenciosa)
  - Guardar schema_name y schema cargado como atributos
```

**Método principal: `analyze`**

```
analyze(self, conversation: str, context: dict | None = None) -> AnalysisResult
```

**Pipeline paso a paso:**

1. **Generar prompt:**
   ```python
   prompt = build_prompt(self.schema, conversation, context)
   ```

2. **Llamar al LLM:**
   ```python
   llm_response = self.llm_client.analyze(prompt)
   ```

3. **Verificar éxito del LLM:**
   ```python
   if not llm_response.success or llm_response.parsed_json is None:
       # Retornar AnalysisResult con fields vacío y error
       return AnalysisResult(
           fields={},
           ...
           validation_errors=[f"LLM failed: {llm_response.error or 'Could not parse JSON'}"]
       )
   ```

4. **Validar resultado contra schema:**
   ```python
   is_valid, errors = self.schema_manager.validate_result(self.schema_name, llm_response.parsed_json)
   # Guardar errors pero NO abortar — retornar el resultado con las advertencias
   ```

5. **Generar embedding** (del campo summary si existe):
   ```python
   embedding = None
   if self.embedding_generator and llm_response.parsed_json.get("summary"):
       embedding = self.embedding_generator.generate(llm_response.parsed_json["summary"])
   ```

6. **Calcular auto-fields:**
   ```python
   import hashlib
   from datetime import datetime, timezone
   
   analyzed_at = datetime.now(timezone.utc).isoformat()
   session_length = conversation.count("\n") + 1  # Aproximación por líneas
   conversation_hash = hashlib.sha256(conversation.encode()).hexdigest()[:16]
   ```

7. **Construir y retornar AnalysisResult:**
   ```python
   return AnalysisResult(
       fields=llm_response.parsed_json,
       schema_name=self.schema_name,
       schema_version=self.schema["metadata"]["version"],
       analyzed_at=analyzed_at,
       analysis_model=self.llm_client.model,
       embedding_model="text-embedding-3-small" if embedding else None,
       session_length=session_length,
       conversation_hash=conversation_hash,
       embedding=embedding,
       llm_response=llm_response,
       validation_errors=errors
   )
   ```

**Función de conveniencia: `analyze` (module-level)**

```python
def analyze(conversation: str,
            schema: str = "universal",
            provider: str = "anthropic",
            model: str | None = None,
            context: dict | None = None,
            enable_embeddings: bool = True) -> AnalysisResult:
    """Función de conveniencia para análisis rápido.
    
    Uso:
        from analyxa import analyze
        result = analyze("User: hello\nAgent: hi", schema="support")
    """
    analyzer = Analyzer(
        schema_name=schema,
        provider=provider,
        model=model,
        enable_embeddings=enable_embeddings
    )
    return analyzer.analyze(conversation, context=context)
```

## NOTAS PARA CLAUDE CODE

1. **AnalysisResult.to_dict() y to_flat_dict() son interfaces clave.** to_dict() es para almacenamiento (Qdrant), to_flat_dict() es para display/export. Los sinks usarán to_flat_dict(). El Qdrant sink futuro usará to_dict().
2. **EmbeddingGenerator NUNCA falla.** Si no hay API key, retorna None. Si la llamada falla, retorna None. El pipeline no se rompe por embeddings. Esto es crítico para que el motor funcione sin OpenAI configurado.
3. **El summary es lo que se vectoriza.** No la conversación completa. El summary es el campo que mejor captura la semántica de la sesión en un texto conciso.
4. **session_length es una aproximación.** Cuenta líneas, no mensajes parseados. Es suficiente para metadata. El parser real de mensajes está en FileSource.read_messages().
5. **La función `analyze()` module-level es para onboarding rápido.** Un developer nuevo puede hacer `from analyxa import analyze; result = analyze(text)` sin saber nada de la arquitectura interna.
6. **Todos los tests usan mocks.** NO llamar APIs reales. Mockear LLMClient.analyze() y EmbeddingGenerator.generate(). Los tests de integración real vendrán en el dogfooding (Fase 2.3).
7. **Los __init__.py de sources/ y sinks/ pueden estar vacíos.** Solo necesitan existir para que Python trate los directorios como packages.
