## CONTEXTO

El Schema System (IF-001) carga schemas YAML. El Prompt Generator (IF-002) convierte schema + conversación en prompts. Ahora necesitamos el componente que **envía esos prompts a un LLM y parsea la respuesta JSON**.

El LLM Client es la interfaz de Analyxa con los modelos de IA. Abstrae la diferencia entre providers (Anthropic, OpenAI) detrás de una interfaz unificada. Recibe un par `{system, user}` y retorna JSON parseado + metadata de uso.

**Diseño multi-provider:** Anthropic (Claude Sonnet) es el provider primario. OpenAI (GPT-4o) es secundario/compatible. La abstracción permite agregar más providers en el futuro sin cambiar el pipeline.

**Parser JSON robusto:** Los LLMs a veces devuelven JSON con code fences (```json...```), texto antes/después del JSON, o JSON ligeramente malformado. El parser debe manejar todos estos casos.

**Conexión con lo anterior:** Consume output de `build_prompt()` (dict con keys "system" y "user").
**Conexión con lo siguiente:** El Analyzer Pipeline (IF-004) orquestará schema → prompt → LLM client → validate → embed.

**Dataclass: `LLMResponse`**

```python
from dataclasses import dataclass, field
from typing import Any

@dataclass
class LLMResponse:
    raw_text: str              # Respuesta cruda del LLM (string completo)
    parsed_json: dict | None   # JSON parseado, None si el parsing falló
    model: str                 # Modelo usado (ej: "claude-sonnet-4-20250514")
    provider: str              # "anthropic" o "openai"
    input_tokens: int          # Tokens de entrada consumidos
    output_tokens: int         # Tokens de salida consumidos
    latency_ms: float          # Tiempo de respuesta en milisegundos
    success: bool              # True si se obtuvo JSON válido
    error: str | None = None   # Mensaje de error si success=False
```

---

**Método: `analyze`**

```
analyze(self, prompt: dict, temperature: float = 0.0, max_tokens: int = 4096) -> LLMResponse
```

**Parámetros:**
- `prompt`: dict con keys "system" y "user" (output de build_prompt)
- `temperature`: 0.0 por default (queremos extracción determinística)
- `max_tokens`: 4096 por default (suficiente para JSON de 18 campos)

**Lógica:**
1. Registrar timestamp de inicio
2. Llamar al provider correspondiente:
   - **Anthropic:** `self._client.messages.create(model=self.model, max_tokens=max_tokens, temperature=temperature, system=prompt["system"], messages=[{"role": "user", "content": prompt["user"]}])`
   - **OpenAI:** `self._client.chat.completions.create(model=self.model, max_tokens=max_tokens, temperature=temperature, messages=[{"role": "system", "content": prompt["system"]}, {"role": "user", "content": prompt["user"]}])`
3. Registrar timestamp de fin
4. Extraer texto de respuesta:
   - Anthropic: `response.content[0].text`
   - OpenAI: `response.choices[0].message.content`
5. Extraer usage:
   - Anthropic: `response.usage.input_tokens`, `response.usage.output_tokens`
   - OpenAI: `response.usage.prompt_tokens`, `response.usage.completion_tokens`
6. Parsear JSON con `_parse_json(raw_text)`
7. Construir y retornar LLMResponse

**Manejo de errores:**
- Si la llamada al API falla (cualquier excepción): capturar, retornar LLMResponse con success=False, error=str del error, parsed_json=None
- NO re-raise excepciones. El caller decide qué hacer con un response fallido.
- Incluir en error el tipo de excepción y mensaje (ej: "anthropic.APIConnectionError: Connection refused")

**Método privado: `_parse_json`**

```
_parse_json(self, text: str) -> dict | None
```

Parser JSON robusto. Intenta extraer JSON válido de la respuesta del LLM.

**Estrategia de parsing (en orden, primer éxito gana):**

1. **Directo:** `json.loads(text.strip())` — el LLM devolvió JSON limpio
2. **Code fences:** Si text contiene ` ```json ` o ` ``` `, extraer contenido entre fences y parsear
3. **Buscar primer `{` y último `}`:** Extraer substring y parsear — el LLM puso texto antes/después del JSON
4. **Si nada funciona:** retornar None

```python
import json
import re

def _parse_json(self, text: str) -> dict | None:
    text = text.strip()
    
    # Estrategia 1: JSON directo
    try:
        return json.loads(text)
    except json.JSONDecodeError:
        pass
    
    # Estrategia 2: Code fences
    patterns = [
        r'```json\s*\n?(.*?)\n?\s*```',
        r'```\s*\n?(.*?)\n?\s*```',
    ]
    for pattern in patterns:
        match = re.search(pattern, text, re.DOTALL)
        if match:
            try:
                return json.loads(match.group(1).strip())
            except json.JSONDecodeError:
                continue
    
    # Estrategia 3: Primer { ... último }
    first_brace = text.find('{')
    last_brace = text.rfind('}')
    if first_brace != -1 and last_brace > first_brace:
        try:
            return json.loads(text[first_brace:last_brace + 1])
        except json.JSONDecodeError:
            pass
    
    # Nada funcionó
    return None
```

## NOTAS PARA CLAUDE CODE

1. **Imports condicionales son ESENCIALES.** No hacer `import anthropic` al top level sin try/except. Un usuario puede tener solo OpenAI instalado y vice versa. El import falla silenciosamente y se valida en __init__.
2. **temperature=0.0 por default.** Queremos extracción determinística, no creatividad. El caller puede subir la temperatura si quiere.
3. **Nunca re-raise excepciones de API.** El método analyze() SIEMPRE retorna LLMResponse. success=False + error message si algo falló. El Analyzer Pipeline decide qué hacer (retry, skip, abort).
4. **El parser JSON es orden de prioridad.** Primero intenta directo (más rápido), luego code fences, luego brute force con {}. No mezclar estrategias.
5. **Los mocks de tests son COMPLETOS.** Mockear desde el SDK (anthropic.Anthropic, openai.OpenAI), no desde internals. Los tests deben verificar el flujo completo: init → analyze → response, sin tocar internet.
6. **No instalar python-dotenv todavía.** Usar `os.environ` directo. La carga de .env la maneja config.py que viene en IF futura (Fase 2).
7. **latency_ms se calcula con time.monotonic()**, no time.time(). monotonic() es inmune a ajustes del reloj del sistema.
8. **El modelo default de Anthropic es "claude-sonnet-4-20250514".** No usar "claude-3-sonnet" ni otros modelos viejos.
